Resources 是 HpAgent 的外部资源访问层，负责管理模型 API 凭据、多模型 HTTP 调用、退避链调度以及 Embedding 向量化。 对外暴露 IResources 抽象接口，上层通过 ResourcePool 统一调度模型调用。

文件目录树

src/resources/
├── __init__.py # 模块导出 & 文档
├── credentials.py # 凭据加密存储 & 模型端点管理
├── model_client.py # 单模型 HTTP 客户端 (Anthopic + OpenAI)
├── resource_pool.py # 多模型调度池 & 退避链
└── embedding.py # Embedding 向量化客户端

src/common/
└── interfaces.py # IResources ABC 定义

interfaces.py — ABC 抽象接口

IResources

common/interfaces.py:19

外部资源访问的统一抽象，ResourcePool 是其唯一实现。上层代码只依赖此接口。

async initialize_models() → None
Worker 启动时加载所有端点并注册到客户端池
async configure_fallback_group(group_name, model_ids) → None
配置退避链：有序模型 ID 列表，失败时按序切换
async generate(messages, model_selector, tools, stream) → Any
模型调用入口。model_selector 可以是退避组名或具体 model_id

ISandbox & IChannel

同文件中还定义了沙箱接口和渠道接口，与 Resources 模块无直接耦合，此处不展开。

CredentialManager class

API 密钥加密存储 + 端点列表管理，线程安全 (RLock)

公开方法
sync register_model_chain(endpoints: List[ModelEndpoint])
注册有序端点列表（清空旧配置 + 加密存储 api_key + 保存脱敏副本）
sync get_model_endpoint_list() → List[ModelEndpoint]
返回解密后完整端点列表，保持注册顺序，供 ResourcePool 退避链使用
sync get_credential(resource_id) → Optional[Credential]
按 resource_id 查询凭据（不解密）
sync get_decrypted_credential(resource_id) → Optional[str]
按 resource_id 查询并解密凭据明文
内部方法
_register_credential(resource_id, credential_type, value, scope, expires_at, metadata)
_encrypt(value) / _decrypt(value) — 占位实现

resource_pool.py — 多模型调度池 & 退避链

ResourcePool class (implements IResources)

状态字段
_credential_manager: CredentialManager
_model_clients: Dict[str, {"client": ModelClient, "priority": int}]
_fallback_groups: Dict[str, List[str]] # group → [model_id, ...]
生命周期方法
async initialize_models() → None
从 CredentialManager 加载端点 → 创建 ModelClient → 构建默认退避组
退避链配置
sync configure_fallback_group(group_name, model_ids)
批量配置退避链（仅此方法被外部调用）。Worker 调用时传入 "chat" / "default" 等分组。
核心调用
async generate(messages, model_selector, tools, stream) → ModelResponse
退避逻辑：
① 解析 model_selector → 退避组或单模型 ID
② 按序尝试候选模型
③ ModelAPIError / ConnectionError / TimeoutError → 记录降级日志，尝试下一个
④ 其他异常（TypeError 等）→ 直接抛出，不隐藏
⑤ 全部失败 → 抛出 ModelAPIError

退避链示例

# Worker 配置
resource_pool.configure_fallback_group("chat", [
    "anthropic:claude-sonnet-4-6",
    "openai:gpt-4o",
])
resource_pool.configure_fallback_group("default", all_ids)

# 调用
response = await pool.generate(
    messages=[{"role":"user","content":"..."}],
    model_selector="chat",    # 使用 chat 退避组
    tools=[...],
)

embedding.py — Embedding 向量化客户端

EmbeddingClient class

OpenAI 兼容的轻量级 embedding 客户端。独立于 ResourcePool 体系，由 retriever 工具直接使用。

_base_url, _api_key, _model, _timeout
sync embed_texts(texts: List[str]) → List[List[float]]
同步入口 — 内部做 async bridge（检测 event loop 状态）
async _embed_async(texts) → List[List[float]]
调用 OpenAI 兼容的 POST /v1/embeddings 端点

create_embedding_client 工厂函数

从 ModelsConfig 数据类创建 EmbeddingClient。解析 embedding 退避链首项 + provider 配置 + ${ENV_VAR} 占位符。

Worker → create_embedding_client(config.models) → EmbeddingClient
                          │
                          ├─ models_config.embedding[0]   → provider, model, timeout
                          └─ models_config.providers[name] → base_url, api_key
