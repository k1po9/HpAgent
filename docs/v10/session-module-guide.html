store.py — 会话存储核心（428 行）

模块中最复杂的文件。内部封装 Redis + Hindsight + 文件备份三个后端，所有方法在底层不可用时静默降级。

class SessionStore:
# ── 构造 ──
__init__(redis_cache, hindsight_client, file_store)
# ── Session lifecycle (4) ──
async create_session(session_id, account_id, channel_type, metadata) → Session
async get_session(session_id) → Session | None
async get_active_session_id(account_id) → str | None
async update_status(session_id, status: SessionStatus) → None

async archive(session_id) → None  ← 标记 COMPLETED + 清理活跃指针 + 最终备份
# ── Events (2) ──
async append_events(session_id, *events: Event) → int
async get_events(session_id, limit=40, offset=0) → list[Event]
# ── Memory (3) ──
async recall_memories(query, account_id, ...) → (items, formatted)
async retain_memories(events, account_id, session_id, ...) → int
async reflect(account_id) → int
# ── Internal (2) ──
async _record_memory_event(session_id, event_type, content) → None
async _backup_to_file(session_id, session, events, is_final) → None
方法类别降级策略
create_session(sid,aid,ch,meta)lifecycleRedis 失败→内存 dict 回退
get_session(sid)lifecycleRedis 失败→内存 dict 回退
get_active_session_id(aid)lifecycleRedis 失败→内存 dict 回退
update_status(sid, status)lifecycle同步更新 Redis + 内存
archive(sid)lifecycle标记 COMPLETED + 清理活跃指针 + 写入最终备份
append_events(sid, *events)eventsRedis RPUSH 失败→仅内存
get_events(sid, limit, offset)eventsRedis LRANGE 失败→仅内存
recall_memories(...)memoryHindsight 不可用→返回空列表
retain_memories(...)memoryHindsight 失败→仅文件备份
reflect(aid)memoryHindsight 不可用→返回 0
Redis Key 设计:
session:{session_id}:events    → List   事件流（RPUSH 追加 / LRANGE 读取）
session:{session_id}:meta      → Hash   会话元数据（Session.to_dict()）
account:{account_id}:active    → String 指向活跃 session_id

TTL: 86400s (24h)
回退: 内存 dict（_mem_events / _mem_sessions / _mem_active）

db.py — SQLite 元数据存储

每个操作新建连接（线程安全），WAL 模式。2 张表，2 个公共方法。

class WorkspaceDB:
# ── 构造 ──
__init__(db_path: str)
# ── 内部 (2) ──
_connect() → sqlite3.Connection
_ensure_schema() → None
# ── 公开 (2) ──
upsert_user(user_uuid, username, profile_path, persistent_dir) → None
insert_session(session: Session) → None
线程安全：每方法内调用 _connect() 创建新连接，操作完立即关闭。
WAL 模式支持多线程并发读，写入通过 5 秒超时重试。 
-- users 表
CREATE TABLE users (
    uuid           TEXT PRIMARY KEY,
    username       TEXT DEFAULT '',
    profile_path   TEXT DEFAULT '',
    persistent_dir TEXT DEFAULT '',
    created_at     TEXT DEFAULT (datetime('now'))
);

-- sessions 表
CREATE TABLE sessions (
    session_id       TEXT PRIMARY KEY,
    user_uuid        TEXT REFERENCES users(uuid) ON DELETE CASCADE,
    status           TEXT DEFAULT 'active',
    task_summary     TEXT DEFAULT '',
    session_dir      TEXT DEFAULT '',
    plan_file        TEXT DEFAULT '',
    conversation_fileTEXT DEFAULT '',
    output_dir       TEXT DEFAULT '',
    tags             TEXT DEFAULT '[]',
    created_at       TEXT DEFAULT (datetime('now')),
    updated_at       TEXT DEFAULT (datetime('now'))
);

CREATE INDEX idx_sessions_user_created
    ON sessions(user_uuid, created_at DESC);
