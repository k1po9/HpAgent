# HpAgent 项目流程时序图

## 1. 启动流程

```mermaid
sequenceDiagram
    participant main as main.py
    participant cfg as AppConfig
    participant worker as Worker
    participant pool as ResourcePool
    participant hrn as HarnessRunner
    participant tmp as Temporal

    main->>cfg: AppConfig.from_yaml("config/config.yaml")
    cfg->>cfg: 加载 config.yaml → 各子 dataclass
    cfg->>cfg: ModelsConfig.from_yaml("config/models.yaml")
    cfg->>cfg: _apply_env_overrides() 环境变量覆盖
    cfg-->>main: AppConfig 实例

    main->>worker: start_worker(config)
    worker->>worker: init_dependencies(config)

    Note over worker: === 依赖组装（严格顺序）===
    worker->>pool: ResourcePool(credential_manager)
    worker->>pool: await initialize_models() — 注册所有模型客户端
    worker->>pool: configure_fallback_group("chat", [...]) — 退避链

    Note over worker: SandboxManager / WorkspaceManager / HindsightClient
    Note over worker: SessionStore / ContextBuilder / AccountService / ChannelRouter

    opt agent.mode == "multi"
        worker->>worker: 加载 config/agents.yaml
        worker->>worker: 创建 MultiAgentExecutor(pool, agents_config, strategy)
    end

    worker->>hrn: HarnessRunner(session_store, ctx_builder, pool, sandbox, router, agent_mode, executor)
    worker->>tmp: inject(harness=harness_runner) — 注入 activities 模块单例
    worker->>tmp: Client.connect(temporal_host)
    worker->>tmp: Worker(client, task_queue, workflows=[OrchestrationWorkflow], activities=[...])
    worker->>worker: channel_router.register(NAPCAT, NapCatChannel())
    worker->>worker: napcat.start_monitor(handle_message) + await asyncio.Future()
    Note over worker: 永久运行，等待消息
```

## 2. 单Agent模式 — 消息处理全链路

```mermaid
sequenceDiagram
    participant QQ as QQ消息
    participant NapCat as NapCatChannel
    participant Handler as handle_message()
    participant TmpWf as OrchestrationWorkflow
    participant Activity as process_turn_activity
    participant HR as HarnessRunner
    participant Session as SessionStore
    participant Ctx as ContextBuilder
    participant Pool as ResourcePool
    participant LLM as LLM API
    participant Sandbox as SandboxManager
    participant Channel as ChannelRouter

    QQ->>NapCat: OneBot JSON 消息
    NapCat->>Handler: UnifiedMessage

    Handler->>Handler: account_service.resolve(ch_type, sender_id)
    Handler->>Handler: workspace_manager.ensure_user() + create_session()
    Handler->>Handler: sandbox_manager.create_session_sandbox()

    Handler->>TmpWf: client.start_workflow(OrchestrationWorkflow.run, user_message, id=session_id)
    Note over TmpWf: Temporal 持久化工作流启动

    TmpWf->>Activity: execute_activity("process_turn_activity", args=[user_message])
    Activity->>HR: _harness.process_turn(user_message)

    Note over HR: === process_turn() 开始 ===

    HR->>Session: _ensure_session() — 幂等创建会话
    HR->>Session: append_events(USER_MESSAGE) — Redis RPUSH
    HR->>Session: get_events(session_id, limit=100) — Redis LRANGE

    loop agentic loop (最多 20 轮)
        HR->>Session: recall_memories(query, account_id, session_id) → Hindsight
        Session-->>HR: ([MemoryItem], formatted_text)

        HR->>Ctx: _build_context(events, channel_type, memories_text)
        Ctx->>Ctx: 组装 system_prompt (身份+风格+环境+记忆+上下文文件)
        Ctx->>Ctx: 过滤事件 → 截断最近 40 条 → 转换为 [{role, content}]
        Ctx-->>HR: List[Dict] — LLM messages 格式

        HR->>Sandbox: _get_tools() — 收集活跃沙箱的全部工具定义
        Sandbox-->>HR: [{type: "function", function: {...}}]

        HR->>Pool: generate(model_selector="chat", messages=context, tools=tools)
        Pool->>LLM: HTTP POST → ModelClient.send()
        LLM-->>Pool: ModelResponse(content, tool_calls, stop_reason)
        Pool-->>HR: ModelResponse

        HR->>Session: append_events(MODEL_MESSAGE)

        alt 有工具调用
            loop 每个 tool_call
                HR->>Sandbox: _execute_tool(name, arguments)
                Sandbox->>Sandbox: sandbox.execute() → nsjail 隔离执行
                Sandbox-->>HR: {output, error}
                HR->>Session: append_events(TOOL_RESULT)
            end
            Note over HR: continue — 下一轮让模型看到工具结果
        else 无工具调用
            Note over HR: break — 本轮结束
        end
    end

    HR->>Channel: _send_response(final_content, user_message)
    Channel->>NapCat: send(UnifiedMessage) → 回复用户
    HR->>Session: retain_memories(turn_events, account_id, session_id) → Hindsight + JSONL
    HR-->>Activity: {content, turns, session_id, account_id}

    Activity-->>TmpWf: result.get("content", "")

    loop 等待后续消息
        TmpWf->>TmpWf: await wait_condition(pending_messages or completed)
        alt 收到 new_message 信号
            TmpWf->>Activity: execute_activity("process_turn_activity", ...)
            Activity->>HR: process_turn(next_message)
            Note over HR: 同上 agentic loop
        else 收到 cancel_session 信号
            Note over TmpWf: 退出循环
        end
    end

    TmpWf->>Activity: execute_activity("archive_session_activity", session_id)
    Activity->>Session: archive(session_id) — 标记完成 + 清理
    TmpWf-->>Handler: {status: "completed", content, turns, ...}
```

## 3. 多Agent模式 — 消息处理全链路

```mermaid
sequenceDiagram
    participant TmpWf as OrchestrationWorkflow
    participant Activity as process_turn_activity
    participant HR as HarnessRunner
    participant Session as SessionStore
    participant Executor as MultiAgentExecutor
    participant Orch as Orchestrator
    participant Strategy as SupervisorControlStrategy
    participant Planner as RealLLMPlanner
    participant Pool as ResourcePool
    participant LLM as LLM API
    participant Agent1 as LLMAgent(default)
    participant Agent2 as LLMAgent(analyst)
    participant Reviewer as RealLLMReviewer

    TmpWf->>Activity: execute_activity("process_turn_activity", ...)
    Activity->>HR: process_turn(user_message)

    Note over HR: === 共享部分（与单Agent相同）===
    HR->>Session: _ensure_session() + append_events(USER_MESSAGE) + get_events()
    HR->>Session: recall_memories() → Hindsight

    Note over HR: === 多Agent分支 ===
    HR->>Executor: execute(goal, history_events, memories_text)

    Note over Executor: 1. 构建 ExecutionContext
    Note over Executor: 2. _build_orchestrator()

    Executor->>Executor: 创建 LLMAgent × N (从 agents.yaml)
    Executor->>Executor: ResourcePoolAdapter(pool, "chat") — CallLLM 适配
    Executor->>Planner: RealLLMPlanner(call_llm)
    Executor->>Reviewer: RealLLMReviewer(call_llm, max_rounds)
    Executor->>Strategy: SupervisorControlStrategy(planner, reviewer)
    Executor->>Orch: Orchestrator(strategy, registry, bus)

    Note over Executor: 3. orchestrator.run(goal, context)

    Orch->>Strategy: initialize_plan(goal, context)
    Strategy->>Planner: plan(goal, context)
    Planner->>Pool: generate(messages, model_selector="chat")
    Pool->>LLM: HTTP POST — 请求任务分解
    LLM-->>Pool: JSON: {tasks: [{task_id, goal, required_tags, depends_on}]}
    Pool-->>Planner: 原始 JSON 字符串
    Planner->>Planner: json.loads() → 解析为 Task 列表 + 依赖图
    Planner-->>Strategy: (tasks, dependencies)
    Strategy-->>Orch: ExecutionPlan(tasks, dependencies)

    loop 编排循环 (while pending)
        Orch->>Strategy: get_ready_batch(results, plan, pending)
        Strategy-->>Orch: [Task, Task] — 依赖已满足的就绪任务

        par 并行批处理
            Orch->>Agent1: execute(task_1, context)
            Agent1->>Pool: generate(messages, "chat", tools)
            Pool->>LLM: HTTP POST
            LLM-->>Agent1: ModelResponse
            Agent1-->>Orch: TaskResult(COMPLETED, output)

            Orch->>Agent2: execute(task_2, context)
            Agent2->>Pool: generate(messages, "chat", tools)
            Pool->>LLM: HTTP POST
            LLM-->>Agent2: ModelResponse
            Agent2-->>Orch: TaskResult(COMPLETED, output)
        end

        Note over Orch: 机械处理: 记录结果 + handoff 注入

        Orch->>Strategy: on_batch_completed(results, plan, context)
        Strategy->>Reviewer: review(results, context)
        Reviewer->>Pool: generate(messages, model_selector="chat")
        Pool->>LLM: HTTP POST — 请求审查结果
        LLM-->>Pool: JSON: {is_done: true/false, new_tasks: [...]}
        Pool-->>Reviewer: 原始 JSON 字符串
        Reviewer-->>Strategy: (is_done, new_tasks)
        Strategy-->>Orch: BatchOutcome(should_terminate, new_tasks, ...)

        alt should_terminate
            Note over Orch: 退出循环
        end
    end

    Orch-->>Executor: dict[str, TaskResult]

    Note over Executor: 4. _synthesize(goal, results) — 合成最终响应
    Executor->>Pool: generate(synthesis_messages, "chat")
    Pool->>LLM: HTTP POST — 请求结果合成
    LLM-->>Executor: 合成后的自然语言回复
    Executor-->>HR: (final_content, turns_taken)

    Note over HR: === 共享部分（与单Agent相同）===
    HR->>Session: append_events(MODEL_MESSAGE)
    HR->>Session: retain_memories() + _send_response()
    HR-->>Activity: {content, turns, session_id, account_id}
    Activity-->>TmpWf: result.get("content")
```

## 5. 阅读代码的推荐顺序

按数据流从外到内、从简到繁：

| 顺序 | 文件 | 看什么 |
|------|------|--------|
| 1 | `src/main.py` | 入口 — 加载配置 → 启动 Worker |
| 2 | `config/config.yaml` | 理解可配置项 |
| 3 | `src/orchestration/config.py` | 配置 dataclass 层次结构 |
| 4 | `src/agent/types.py` | 核心数据类型 (Task / TaskResult / ExecutionPlan / BatchOutcome) |
| 5 | `src/agent/interfaces.py` | 5个抽象基类 (BaseAgent / ControlStrategy / AgentRegistry / MessageBus) |
| 6 | `src/agent/context.py` | ExecutionContext + SharedMemory |
| 7 | `src/agent/llm_agent.py` | 最简 Agent 实现 — 理解 Agent 如何执行任务 |
| 8 | `src/agent/factory.py` | 三个 build_* 工厂函数 — 理解编排器如何组装 |
| 9 | `src/agent/strategies.py` | 三种 Strategy (Supervisor/Council/Workflow) + RealLLM* |
| 10 | `src/agent/orchestrator.py` | 纯机械调度循环 — 并行批处理 + 超时 + handoff |
| 11 | `src/agent/runner.py` | MultiAgentExecutor — 多Agent执行器（桥接 HarnessRunner） |
| 12 | `src/harness/runner.py` | HarnessRunner — 单Agent/多Agent 统一入口 |
| 13 | `src/orchestration/worker.py` | 依赖组装 + Temporal Worker 启动 |
| 14 | `src/orchestration/workflow.py` | Temporal Workflow 编排层 |

## 6. 关键数据流

```
用户消息
  → UnifiedMessage (渠道层)
  → user_message dict (Worker)
  → Temporal Workflow 输入
  → HarnessRunner.process_turn()
      ├── [单Agent] ReAct 循环
      │     Event → ContextBuilder → [{role, content}] → ResourcePool.generate()
      │     → ModelResponse → Event → tools → ... → 回复
      │
      └── [多Agent] MultiAgentExecutor.execute()
            user_content → LLM(Planner) → ExecutionPlan(tasks)
            → 并行 LLMAgent × N → dict[str, TaskResult]
            → LLM(Reviewer) → BatchOutcome → ...
            → LLM(Synthesizer) → 回复
  → ChannelRouter.send(UnifiedMessage)
  → 用户看到回复
```
