Sandbox 模块架构

Sandbox 是 HpAgent 的"手"层（Hands Layer），负责在隔离的 workspace 环境中执行模型决策的工具调用。在"手脑分离"架构中，编排层（大脑）决策调用哪个工具，沙箱层（手）接收 workspace 路径并按类别路由执行。

设计原则

原则说明
Workspace 绑定Sandbox 从外部接收 workspace 路径；所有本地工具通过闭包绑定该路径，在指定目录下执行文件操作
类别路由3 类工具各有执行路径：native 进程内执行、mcp 远端调用、skill 流水线展开
可选加固Nsjail 仅对 Bash 工具启用，其他本地工具始终进程内执行。默认关闭，通过配置开关控制
Per-Session 隔离每个会话拥有独立的 Sandbox + ToolRegistry，本地工具绑定到专属 workspace 路径
路径穿越防护所有用户提供的路径通过 safe_resolve() 验证，确保不越出 workspace 根目录

模块总览

关键数据流

1. Worker 收到消息 → SandboxManager.create_session_sandbox() 创建 per-session 沙箱
2. HarnessRunner.process_turn() → 模型决策调用工具
3. HarnessRunner._execute_tool() → SandboxManager.get_sandbox_for_session(session_id)
4. Sandbox.execute(tool_name, arguments) → 查 ToolRegistry.get_category()
5. native 工具 → tool.ainvoke() 进程内执行（workspace 已绑定）
6. Bash 工具 + nsjail 启用 → NsjailExecutor.execute() 子进程加固
7. 模型看到结果 → 继续推理 或 发送回复

6 个本地工具 (Native Local Tools)

每个工具通过工厂函数 create_*_tool(workspace_root: str) → StructuredTool 创建。工厂函数接收 workspace 根路径，通过闭包将路径绑定到工具内部，确保所有文件操作都在指定 workspace 内执行。

工具名参数描述类别
fs_read path: str
offset: int = 1
limit: int? = None		 读取文件并返回带行号的文本。每行格式为 行号\t内容。文件超过 2000 行时默认仅显示前 500 行，可使用 offset/limit 翻页。文件不存在时提示用 Glob 或检查路径。 native
fs_write path: str
content: str		 创建或完全覆盖文件。自动创建中间目录。返回 {"bytes_written": N}。建议用 fs_edit 做局部修改。 native
fs_edit ★ path: str
old_string: str
new_string: str		 核心工具。精确字符串替换，遵循 4 条规则：① 文件必须存在 ② old_string 为空则追加到文件末尾 ③ old_string 必须恰好匹配一次（含空白字符）④ 匹配零次或多次均报错，多次时列出匹配行号。返回 {"replacements": 1}。 native
Glob root: str
pattern: str		 文件名模式匹配。返回相对路径列表，按修改时间降序排列。支持递归通配 **/*.py。最多返回 200 条结果，超出时标记 truncated。 native
Grep root: str
query: str
glob: str? = None
regex: bool = False		 文件内容搜索。默认字面量匹配，设置 regex=True 启用正则。返回 [{path, line, col, content}]。最多 50 条结果，超出截断。可用 glob 参数过滤文件类型。 native
Bash cmd: str
cwd: str = ""
timeout: int = 120		 在 workspace 内执行 Shell 命令。安全约束：cwd 限制在 workspace 内、拦截 rm -rf 等危险模式、stdout 截断至 50KB、最大 timeout 300s。nsjail 启用时自动走 nsjail 加固。 native

共享工具函数

def safe_resolve(workspace_root: str, user_path: str) → str
将用户提供的相对路径解析为 workspace 下的绝对路径，阻止路径穿越攻击（如 ../../../etc/passwd）。
def safe_cwd(workspace_root: str, cwd: str) → str
解析 Bash 工作目录，确保不越出 workspace。空 cwd 默认返回 workspace 根路径。
def make_relative(workspace_root: str, abs_path: str) → str
将 workspace 下的绝对路径转为相对路径，用于工具输出。

工具执行流程

从模型发起工具调用到沙箱执行完毕，完整的调用链路：

类别路由规则

类别执行方式说明
native tool.ainvoke() 进程内 本地工具通过闭包已绑定 workspace 路径，直接调用即可。仅 Bash 工具可选走 nsjail 子进程加固。
mcp MCP 远端调用 通过 MCPToolManager 管理的 HTTP/SSE 会话发送 JSON-RPC 请求到远端 MCP 服务器执行。
skill SkillPipeline 展开 Skill 定义为多步流水线（steps），每个 step 调用 registry 中的其他工具，逐步执行并聚合结果。
